  2122ŌåÆ### Problem
  2123ŌåÆThe crypto wallets system was running on top of a hardcoded `DEMO_CRYPTO_WALLETS` array (and a parallel `DEMO_CRYPTO_BALANCES` in-memory map in the cashout route) ŌĆö meaning:
  2124ŌåÆ- Wallet balances were not persisted to the DB; a server restart reset them.
  2125ŌåÆ- The cashout/swap/trade routes mutated in-memory JS objects instead of writing to the `Wallet` table, so concurrent route handlers and the wallets UI were out of sync.
  2126ŌåÆ- No transactional safety: a cashout that succeeded at the crypto-debit step but failed mid-way through the fiat-credit would leave the system in an inconsistent state.
  2127ŌåÆ
  2128ŌåÆ### Solution ŌĆö 5 deliverables
  2129ŌåÆ
  2130ŌåÆ#### 1. `prisma/seed-crypto-wallets.ts` (idempotent seed script)
  2131ŌåÆ- Creates 8 crypto `Wallet` rows for the demo user (BTC=0.04582, ETH=1.2847, USDT=2850.50, USDC=1240.00, BNB=3.582, SOL=12.45, PI=1850.0, TRX=4580.0).
  2132ŌåÆ- Each wallet: `userId=DEMO_USER_ID`, `currency=crypto code`, `balance=amount`, `ledgerBalance=amount`, `type="crypto"`, `label="Crypto Wallet"`, `isDefault=false`, `status="active"`.
  2133ŌåÆ- Uses `findFirst` per `(userId, currency, type="crypto")` before `create` ŌĆö re-running the script is a no-op (does not duplicate or overwrite existing rows). Verified by running twice.
  2134ŌåÆ- Ran successfully: `bun run prisma/seed-crypto-wallets.ts` ŌåÆ 8 wallets created on first run, all marked "Ō£ō exists" on second run.
  2135ŌåÆ
  2136ŌåÆ#### 2. `/api/crypto/wallets` (route.ts) ŌĆö fully rewritten to use the DB
  2137ŌåÆ- Queries `db.wallet.findMany({ where: { userId: DEMO_USER_ID, type: "crypto" } })`.
  2138ŌåÆ- **Self-bootstrapping**: if no crypto wallets exist in the DB yet, the route seeds them inline (`createMany`) on the first GET ŌĆö so the system always returns data even on a fresh DB.
  2139ŌåÆ- For each wallet, fetches the real price from CoinGecko via `getCryptoRates()` (60s cache, single-flight).
  2140ŌåÆ- Generates a deterministic deposit address per crypto using `SHA-256(userId + ":" + currency)`, formatted with the correct per-chain prefix (`bc1qŌĆ”` for BTC, `0xŌĆ”` for EVM chains, `bnb1ŌĆ”` for BNB, `TŌĆ”` for TRX, `pi_network_ŌĆ”` for PI, etc).   2141ŌåÆ- Computes `valueUSD = balance ├Ś realPrice` and `valueNGN = valueUSD ├Ś 1535` (NGN per USD from `FIAT_USD_RATE`).

  2145ŌåÆ#### 3. `/api/crypto/cashout` (route.ts) ŌĆö checks real balance, atomic
  2146ŌåÆ- **Before processing**: re-fetches the crypto wallet from the DB inside a `db.$transaction`, checks `balance >= amount`, returns `400 { error: "Insufficient {crypto} balance (available: X)" }` if not.
  2147ŌåÆ- **Inside the transaction**: (a) decrements the crypto wallet by `numericAmount`, (b) finds-or-creates the destination fiat wallet, (c) credits the fiat wallet by `fiatCredited`, (d) creates the paired debit + credit `Transaction` records (with `pairedTxRef` linking both ways).
  2148ŌåÆ- Notifications are created **outside** the financial transaction (so a notification failure can't roll back a successful cashout).
  2149ŌåÆ- The GET quote endpoint now reads `availableBalance` from the DB instead of the old in-memory constant, so the UI's "available" reflects prior cashouts in real time.
  2150ŌåÆ- Removed the old `DEMO_CRYPTO_BALANCES` in-memory map and `DEMO_CRYPTO_CODES` constant entirely.
  2151ŌåÆ
  2152ŌåÆ#### 4. `/api/crypto/swap` (route.ts) ŌĆö checks real balance, atomic
  2153ŌåÆ- **Inside a `db.$transaction`**: (a) re-fetches the `fromCrypto` wallet, checks `balance >= amount` (returns `400 "Insufficient {from} balance (available: X)"` if not), (b) finds-or-creates the `toCrypto` wallet, (c) decrements the source wallet by `amount`, (d) credits the destination wallet by `convertedAmount` (after 0.3% swap fee + network fee), (e) creates the exchange `Transaction` record.
  2154ŌåÆ- Returns the new `remainingFromBalance` and `newToBalance` in the response so the UI can update without a refetch.
  2155ŌåÆ- The GET quote endpoint stays unchanged (price-only, no balance check needed).
  2156ŌåÆ
  2157ŌåÆ#### 5. `/api/crypto/trade` (route.ts) ŌĆö checks real balance, atomic
